queue_new_job(*args, **kwargs)¶

Creates a new job for the given type and data. The new job is immediately placed on the queue. The given job_type model must have already been saved in the database (it must have an ID). The given event model must have already been saved in the database (it must have an ID). The new job, job_exe, and queue models are saved in the database in an atomic transaction. If the data is invalid, a job.configuration.data.exceptions.InvalidData will be thrown.

Parameters:
  • job_type (job.models.JobType) – The type of the new job to create and queue
  • data (dict) – JSON description defining the job data to run on
  • event (trigger.models.TriggerEvent) – The event that triggered the creation of this job
Returns:

The ID of the new job and the ID of the job execution
Return type:

int
Raises job.configuration.data.exceptions.InvalidData:
 

If the job data is invalid
