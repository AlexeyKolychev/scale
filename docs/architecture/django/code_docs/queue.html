queue_new_recipe(*args, **kwargs)¶

Creates a new recipe for the given type and data. and queues any of its jobs that are ready to run. If the new recipe is superseding an old recipe, superseded_recipe, delta, and superseded_jobs must be provided and the caller must have obtained a model lock on all job models in superseded_jobs and on the superseded_recipe model. All database changes occur in an atomic transaction.

Parameters:
  • recipe_type (recipe.models.RecipeType) – The type of the new recipe to create
  • data (recipe.data.recipe_data.RecipeData) – The recipe data to run on, should be None if superseded_recipe is provided
  • event (trigger.models.TriggerEvent) – The event that triggered the creation of this recipe
  • superseded_recipe (recipe.models.Recipe) – The recipe that the created recipe is superseding, possibly None
  • delta (recipe.handlers.graph_delta.RecipeGraphDelta) – If not None, represents the changes between the old recipe to supersede and the new recipe
  • superseded_jobs ({string: job.models.Job}) – If not None, represents the job models (stored by job name) of the old recipe to supersede
Returns:

The ID of the new recipe
Return type:

int

:raises recipe.configuration.data.exceptions.InvalidRecipeData: If the recipe data is invalid
