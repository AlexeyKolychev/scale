job.clock.perform_tick()¶

Performs an iteration of the Scale clock.

A clock iteration consists of inspecting any trigger rules and corresponding events to see if there are any that have exceeded their scheduled time threshold. For rules that are due to run, a new event is created of the configured type and the registered clock function is executed.

job.clock.register_processor(name, processor_class)¶

Registers the given processor class definition to be called by the Scale clock at the given interval.

Processors from other applications can be registered during their ready() method.

Parameters:
  • name (str) – The system name of the processor, which is used in trigger rule configurations.
  • processor_class (job.clock.ClockProcessor) – The processor class to invoke when the associated event is triggered.

class job.models.Job(*args, **kwargs)¶

Bases: django.db.models.base.Model

Represents a job to be run on the cluster. Any status updates to a job model requires obtaining a lock on the model using select_for_update(). If a related job execution model is changing status as well, its model lock must be obtained BEFORE obtaining the lock for the job model.

Parameters:
  • job_type (django.db.models.ForeignKey) – The type of this job
  • job_type_rev (django.db.models.ForeignKey) – The revision of the job type when this job was created
  • status (django.db.models.CharField) – The status of the job
  • event (django.db.models.ForeignKey) – The event that triggered the creation of this job
  • error (django.db.models.ForeignKey) – The error that caused the failure (should only be set when status is FAILED)
  • data (djorm_pgjson.fields.JSONField) – JSON description defining the data for this job. This field must be populated when the job is first queued.
  • results (djorm_pgjson.fields.JSONField) – JSON description defining the results for this job. This field is populated when the job is successfully completed.
  • docker_params (djorm_pgjson.fields.JSONField) – JSON array of 2-tuples (key-value) which will be passed as-is to docker. See the mesos prototype file for further information.
  • priority (django.db.models.IntegerField) – The priority of the job (lower number is higher priority)
  • timeout (django.db.models.IntegerField) – The maximum amount of time to allow this job to run before being killed (in seconds)
  • max_tries (django.db.models.IntegerField) – The maximum number of times to try executing this job in case of errors (minimum one)
  • num_exes (django.db.models.IntegerField) – The number of executions this job has had
  • cpus_required (django.db.models.FloatField) – The number of CPUs required for this job
  • mem_required (django.db.models.FloatField) – The amount of RAM in MiB required for this job
  • disk_in_required (django.db.models.FloatField) – The amount of disk space in MiB required for input files for this job
  • disk_out_required (django.db.models.FloatField) – The amount of disk space in MiB required for output (temp work and products) for this job
  • is_superseded (django.db.models.BooleanField) – Whether this job has been superseded and is obsolete. This may be true while superseded_by_job (the reverse relationship of superded_job) is null, indicating that this job is obsolete (its recipe has been superseded), but there is no new job that has directly taken its place.
  • root_superseded_job (django.db.models.ForeignKey) – The first job in the chain of superseded jobs. This field will be null for the first job in the chain (i.e. jobs that have a null superseded_job field).
  • superseded_job (django.db.models.ForeignKey) – The job that was directly superseded by this job. The reverse relationship can be accessed using ‘superseded_by_job’.
  • delete_superseded (django.db.models.BooleanField) – Whether this job should delete the products of the job that it has directly superseded
  • created (django.db.models.DateTimeField) – When the job was created
  • queued (django.db.models.DateTimeField) – When the job was added to the queue to be run when resources are available
  • started (django.db.models.DateTimeField) – When the job started running
  • ended (django.db.models.DateTimeField) – When the job stopped running, which could be due to successful completion or an error condition
  • last_status_change (django.db.models.DateTimeField) – When the job’s last status change occurred
  • superseded (django.db.models.DateTimeField) – When this job was superseded
  • last_modified (django.db.models.DateTimeField) – When the job was last modified

class job.models.JobExecution(*args, **kwargs)¶

Bases: django.db.models.base.Model

Represents an instance of a job being queued and executed on a cluster node. Any status updates to a job execution model requires obtaining a lock on the model using select_for_update().

Parameters:
  • job (django.db.models.ForeignKey) – The job that is being executed
  • status (django.db.models.CharField) – The status of the run
  • error (django.db.models.ForeignKey) – The error that caused the failure (should only be set when status is FAILED)
  • command_arguments (django.db.models.CharField) – The argument string to execute on the command line for this job execution. This field is populated when the job execution is scheduled to run on a node and is updated when any needed pre-job steps are run.
  • timeout (django.db.models.IntegerField) – The maximum amount of time to allow this job to run before being killed (in seconds)
  • docker_params (djorm_pgjson.fields.JSONField) – JSON array of 2-tuples (key-value) which will be passed as-is to docker. See the mesos prototype file for further information.
  • node (django.db.models.ForeignKey) – The node on which the job execution is being run
  • environment (djorm_pgjson.fields.JSONField) – JSON description defining the environment data for this job execution. This field is populated when the job execution is scheduled to run on a node.
  • cpus_scheduled (django.db.models.FloatField) – The number of CPUs scheduled for this job execution
  • mem_scheduled (django.db.models.FloatField) – The amount of RAM in MiB scheduled for this job execution
  • disk_in_scheduled (django.db.models.FloatField) – The amount of disk space in MiB scheduled for input files for this job execution
  • disk_out_scheduled (django.db.models.FloatField) – The amount of disk space in MiB scheduled for output (temp work and products) for this job execution
  • disk_total_scheduled (django.db.models.FloatField) – The total amount of disk space in MiB scheduled for this job execution
  • requires_cleanup (django.db.models.BooleanField) – Whether this job execution still requires cleanup on the node
  • pre_task_id (django.db.models.CharField) – The unique ID of the pre-task
  • pre_started (django.db.models.DateTimeField) – When the pre-task was started
  • pre_completed (django.db.models.DateTimeField) – When the pre-task was completed
  • pre_exit_code (django.db.models.IntegerField) – The exit code of the pre-task
  • job_task_id (django.db.models.CharField) – The unique ID of the task running the job
  • job_started (django.db.models.DateTimeField) – When the main job task started running
  • job_completed (django.db.models.DateTimeField) – When the main job task was completed
  • job_exit_code (django.db.models.IntegerField) – The exit code of the main job task
  • post_task_id (django.db.models.CharField) – The unique ID of the post-task
  • post_started (django.db.models.DateTimeField) – When the post-task was started
  • post_completed (django.db.models.DateTimeField) – When the post-task was completed
  • post_exit_code (django.db.models.IntegerField) – The exit code of the post-task
  • stdout (django.db.models.TextField) – The stdout contents of the entire job execution. This field should normally be deferred when querying for job executions since it can be large and often is not needed.
  • stderr (django.db.models.TextField) – The stderr contents of the entire job execution. This field should normally be deferred when querying for job executions since it can be large and often is not needed.
  • current_stdout_url (django.db.models.URLField) – URL for gettng the current stdout log contents
  • current_stderr_url (django.db.models.URLField) – URL for gettng the current stderr log contents
  • results_manifest (djorm_pgjson.fields.JSONField) – The results manifest generated by the job’s algorithm
  • results (djorm_pgjson.fields.JSONField) – JSON description defining the results for this job execution. This field is populated when the post-job steps are successfully completed.
  • created (django.db.models.DateTimeField) – When the job execution was created
  • queued (django.db.models.DateTimeField) – When the job was added to the queue for this run and went to QUEUED status
  • started (django.db.models.DateTimeField) – When the job was scheduled and went to RUNNING status
  • ended (django.db.models.DateTimeField) – When the job execution ended (FAILED, COMPLETED, or CANCELED)
  • cleaned_up (django.db.models.DateTimeField) – When the job execution was cleaned up on the node
  • last_modified (django.db.models.DateTimeField) – When the job execution was last modified
